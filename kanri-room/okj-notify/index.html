BTC/JPY 通知設定

GitHub Actions（*/5 分）で OKJ の BTC/JPY を取得し、ntfy.sh 経由で iPhone へ通知します。

読み込み中…

通知先（ntfy.sh）

iPhone 側 ntfy アプリで購読中のトピック。変更時は iPhone 側も再購読が必要です。

英数 / _ / - のみ。6〜64文字。
4 = high（推奨）、5 = max（音強制）。

円幅トリガー（band_yen）

10万円帯のように、帯を跨いだら通知。前回通知価格から帯番号が変わったら発火。

有効化 OFF にすると円幅トリガーは発火しません。
推奨: 100,000（10万円）。範囲: 1,000〜100,000,000。

%幅トリガー（band_percent）

前回通知時の価格から ±X% 動いたら通知。閾値は通知都度に更新（band方式）。

有効化 OFF にすると %幅トリガーは発火しません。
推奨: 1.0%。範囲: 0.1〜50.0。

RSI トリガー（rsi）

指定した時間足の RSI が買われすぎ / 売られすぎゾーンに入った瞬間に通知（ゾーン遷移時のみ発火）。

有効化 OFF にすると RSI トリガーは発火しません。
OKJ API がサポートする粒度のみ。推奨: 1時間足。
標準は 14。範囲: 2〜50 の整数。
2〜50 の整数で入力してください。
推奨: 70。範囲: 50〜100。oversold より大きい値である必要があります。
50〜100 かつ oversold より大きい値で入力してください。
推奨: 30。範囲: 0〜50。overbought より小さい値である必要があります。
0〜50 かつ overbought より小さい値で入力してください。

MACD トリガー（macd）

指定した時間足の MACD とシグナルの交差方向が変わった瞬間に通知（signal_cross 方式）。

有効化 OFF にすると MACD トリガーは発火しません。
OKJ API がサポートする粒度のみ。推奨: 1時間足。
標準は 12。範囲: 2〜50 の整数。slow より小さい値である必要があります。
2〜50 かつ slow より小さい整数で入力してください。
標準は 26。範囲: 3〜100 の整数。fast より大きい値である必要があります。
3〜100 かつ fast より大きい整数で入力してください。
標準は 9。範囲: 2〜50 の整数。
2〜50 の整数で入力してください。
MACD とシグナルの交差方向が変わったら通知（固定）。

Fear & Greed トリガー（fear_greed）

alternative.me の Crypto Fear & Greed Index。極端ゾーンに入った瞬間だけ 1 回通知。

有効化 OFF にすると Fear & Greed トリガーは発火しません。
推奨: 75。範囲: 50〜100。extreme_fear より大きい値である必要があります。
50〜100 かつ extreme_fear より大きい値で入力してください。
推奨: 25。範囲: 0〜50。extreme_greed より小さい値である必要があります。
0〜50 かつ extreme_greed より小さい値で入力してください。
