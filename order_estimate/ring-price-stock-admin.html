リング価格・在庫 管理画面

在庫数・在庫状態・見積価格を確認して修正し、そのまま GitHub 上の公開CSVへ反映するための管理画面です。

見積もり画面へ戻る 正本CSVを開く

新しいリング候補を追加

未登録の規格 / リング種別 / 径 / 色を、全件テーブルを開かずに追加します。たとえば `Mini6 / 通常リング / 20mm / Black` のような行をここから作れます。
巡回結果CSV
公開CSVを再読込 編集をやり直したいときに、今公開されている `ring-price-stock.csv` を読み直します。
CSVダウンロード 編集結果をCSVとして書き出します。バックアップを残したいときや、内容確認用の控えに使います。
GitHubトークン設定 GitHubへ直接反映するときのトークンを、このブラウザのセッション中だけ保持します。
GitHubへ反映 現在のCSV内容を GitHub リポジトリへ直接保存します。GitHub Pages 側のCSVもこの更新で置き換わります。
読込元: 初期化中 状態: 保存済み 行数: 0 GitHub: トークン未設定 価格差分: 0件 巡回: 未実行 最新巡回: 未確認
おすすめの流れは `公開CSVを再読込` → 在庫や価格を修正 → `GitHubへ反映` です。 `CSVダウンロード` はバックアップ用です。公開データを更新する本操作は `GitHubへ反映` です。
この画面ではローカルCSVの読込・保存は使いません。必要なら `CSVダウンロード` でその時点の内容を控えとして残せます。
巡回サーバー待機中です。未起動の場合は `start-ring-price-crawl-server.bat` を実行してください。

在庫クイック検索

この一覧で直した在庫数・在庫状態・見積価格を、そのまま GitHub 上の公開CSVへ反映します。

全件テーブル

正本CSVの全行をここで直接編集できます。価格が反映されないときは、`planner_type / ring_id / color_id / price_jpy / stock_status / stock_qty` をまず確認してください。

巡回結果レビュー

価格差分がある候補だけを一覧表示します。`採用する` は優先ソースと見積価格へ反映し、`不採用` はその候補スロットをクリアします。

新しい価格情報を反映

新たに見つけた価格情報を、ブラウザ上から現在のCSVデータへ取り込みます。保存は `直接上書き保存` または `CSVダウンロード` で確定します。
対象行を選ぶと、ここに現在の優先ソースと価格差分の見込みを表示します。
